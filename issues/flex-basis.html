What is Flex Basis?

When it comes to flex styles it can seem a bit overwhelming at first. Personally the most difficult for me to grasp was flex-basis. flex-basis is the size of a flex item before it is placed into a flex container. That doesn't seem so hard, right?

Well the confusing part is that when flex-basis isn't set, the element will fall back on it's width style and if that isn't set it'll fall back on the content of the element to determine it's width. You can think of an element's width determined in order of the presence of the:

  1. flex-basis style (with optional upper and lower limits set with max-width and min-width)
  2. width style
  3. element's content

So why can't I just use width?

You could but then your flex system would only accommodate rows not columns. Flex-basis accounts for width and height. Remember, the definition above states that flex-basis determines the size of the flex item not just it's width. So for columns, a flex item height is determined by the:

  1. flex-basis style (with optional upper and lower limits set with max-height and min-height)
  2. height style
  3. element's content

Not enough container space

Row

This example is a bit more complex. What happens if a container doesn't have enough space to accommodate each item's flex-basis and flex-wrap is set to nowrap? Then as long as each item has the same flex-shrink property greater then 0, they will scale down to maintain the same aspect ratio. In this example, the first element has a flex-basis of 100px and the other two are set to 200px. SO the ratio is 1:2:2. Since there is only 400px of space to fit the 500px of content, the first gets 80px of width while the other two get 160px in order to maintain the same ratio.
